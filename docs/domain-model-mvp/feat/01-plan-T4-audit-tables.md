# T4: Create Audit and Upload Tables (pliego_uploads, analyses)

## Scope

- `supabase/migrations/20260504000004_audit_tables.sql` — DDL for pliego_uploads and analyses

## Changes

### pliego_uploads table

- `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
- `proceso_id uuid NOT NULL REFERENCES procesos(id) ON DELETE RESTRICT` — RESTRICT: cannot delete a proceso that has uploads
- `uploaded_by_company_id uuid NOT NULL REFERENCES companies(id) ON DELETE RESTRICT` — audit trail must survive company edits
- `file_sha256 text NOT NULL` — SHA-256 hex of raw file bytes; 64-char string
- `file_storage_key text NOT NULL` — Supabase Storage key: `companies/<company_id>/pliegos/<sha256>.pdf`
- `uploaded_at timestamptz NOT NULL DEFAULT now()`
- `uploader_ip text` — nullable in schema; application MUST populate; nullable for seed data
- `declaration_accepted_at timestamptz` — nullable until user accepts declaration
- `declaration_text_version text` — version identifier of the declaration text shown to user
- `status text NOT NULL DEFAULT 'active' CHECK (status IN ('active','flagged','superseded'))` — active = in use; flagged = under review; superseded = replaced by newer upload
- `UNIQUE (proceso_id, uploaded_by_company_id, file_sha256)` — prevents duplicate uploads of same file by same company for same proceso
- `ALTER TABLE pliego_uploads ENABLE ROW LEVEL SECURITY`

### analyses table

- `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
- `proceso_id uuid NOT NULL REFERENCES procesos(id) ON DELETE RESTRICT`
- `company_id uuid NOT NULL REFERENCES companies(id) ON DELETE RESTRICT`
- `pliego_upload_id uuid NOT NULL REFERENCES pliego_uploads(id) ON DELETE RESTRICT`
- `proceso_metadata_snapshot jsonb NOT NULL` — full datos.gov.co response at INSERT time; `'{}'` for `unverified`
- `proceso_lookup_status text NOT NULL CHECK (proceso_lookup_status IN ('verified','unverified','failed'))`
- `verdict text CHECK (verdict IN ('verde','amarillo','rojo'))` — nullable until analysis completes
- `estado text NOT NULL DEFAULT 'pending' CHECK (estado IN ('pending','extracting','analyzing','completed','failed'))`
- `created_at timestamptz NOT NULL DEFAULT now()` — immutable; identifies this analysis run
- `completed_at timestamptz` — nullable; set when `estado = 'completed'`
- `input_tokens int` — nullable; total tokens sent to LLM
- `output_tokens int` — nullable; tokens generated by LLM
- `cached_tokens int` — nullable; tokens served from Anthropic prompt cache
- `cost_usd numeric(10,6)` — nullable; total cost including retries and OCR
- `latency_ms int` — nullable; wall-clock time for the analysis pipeline
- `ALTER TABLE analyses ENABLE ROW LEVEL SECURITY`

### Design Rationale (Audit Completeness)

`pliego_uploads` and `analyses` form the core audit triangle with `procesos`. Separating them from extraction tables (T5) means the audit trail exists even if extraction has never run — enabling future "upload only" flows without schema changes.
